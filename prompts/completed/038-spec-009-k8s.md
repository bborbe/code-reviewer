2. **Create `watcher/github/k8s/github-pr-watcher.yaml`** (Deployment):
   ```yaml
   apiVersion: apps/v1
   kind: Deployment
   metadata:
     name: github-pr-watcher
     namespace: '{{ "NAMESPACE" | env }}'
   spec:
     replicas: 1
     selector:
       matchLabels:
         app: github-pr-watcher
     template:
       metadata:
         labels:
           app: github-pr-watcher
       spec:
         priorityClassName: github-pr-watcher
         containers:
           - name: github-pr-watcher
             image: '{{ "DOCKER_REGISTRY" | env }}/github-pr-watcher:{{ "BRANCH" | env }}'
             env:
               - name: GH_TOKEN
                 valueFrom:
                   secretKeyRef:
                     name: github-pr-watcher
                     key: GH_TOKEN
               - name: KAFKA_BROKERS
                 value: '{{ "KAFKA_BROKERS" | env }}'
               - name: STAGE
                 value: '{{ "STAGE" | env }}'
             resources:
               requests:
                 cpu: 100m
                 memory: 128Mi
               limits:
                 cpu: 100m
                 memory: 128Mi
             volumeMounts:
               - name: cursor
                 mountPath: /data
         volumes:
           - name: cursor
             persistentVolumeClaim:
               claimName: github-pr-watcher
   ```

   Notes:
   - `replicas: 1` — single pod by design (no concurrent-watcher locking required)
   - `priorityClassName: github-pr-watcher` is REQUIRED so the ResourceQuota's PriorityClass scope-selector binds to this pod (without it, the quota matches zero pods and is silently non-binding)
   - CPU/memory are lighter than the pr-reviewer agent (no Claude CLI, no git)
   - `GH_TOKEN` comes from the Secret (sensitive); `KAFKA_BROKERS` and `STAGE` come directly from env vars at template time (non-sensitive — `KAFKA_BROKERS` is a literal cluster URL in `common.env`; `STAGE` comes from `dev.env`/`prod.env`)
   - Image tag uses `'{{ "BRANCH" | env }}'` — the same templating filter used by sibling services. The image registry uses `'{{ "DOCKER_REGISTRY" | env }}'` (defined in `common.env`)
